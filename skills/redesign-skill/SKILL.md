# Redesign Skill Complete Content

This comprehensive guide upgrades existing websites and apps to premium quality by auditing current designs, identifying generic AI patterns, and applying high-end standards without breaking functionality.

## Core Workflow

The skill follows a three-step sequence: **Scan** the codebase to identify frameworks and styling methods, **Diagnose** problems using the detailed audit checklist, then **Fix** issues by improving existing code rather than rewriting.

## Major Audit Categories

**Typography**: Replace generic fonts with characterful options like Geist or Outfit; increase headline presence; limit body text width; use varied font weights (400, 500, 600, 700); and fix issues like oversized widths and orphaned words.

**Color & Surfaces**: Avoid pure black backgrounds, oversaturated accents, and "purple/blue AI gradients." Instead use off-blacks, desaturated colors, tinted shadows, and subtle texture overlays to eliminate sterile flatness.

**Layout**: Break symmetry with asymmetric grids; replace the generic three-card column pattern; use "min-height: 100dvh instead of height: 100vh"; add max-width containers; and align elements consistently across rows.

**Interactivity**: Add hover states, active feedback with transforms, smooth transitions (200-300ms), visible focus rings, skeleton loaders, and empty/error state designs.

**Content**: Use realistic names, organic data (not round numbers), contextual brand names, active voice, and sentence case headers instead of cliches and placeholder text.

**Components**: Remove generic card borders; avoid single accent-color button pairs; replace accordion FAQs with alternatives; and simplify footer link farms.

**Iconography**: Choose icon sets beyond Lucide/Feather; replace cliche metaphors; standardize stroke widths; and always include branded favicons.

**Code Quality**: Use semantic HTML; move inline styles to classes; employ relative units; add alt text; establish z-index scales; and remove dead code.

## Strategic Omissions

The guide highlights what AI typically forgets: legal links, back navigation, custom 404 pages, form validation, skip-links, and cookie consent banners.

## Upgrade Techniques

High-impact improvements include variable font animation, text mask reveals, broken grids, parallax stacks, spring physics motion, glassmorphism borders, and grain overlays.

## Fix Priority

Changes should follow this sequence for maximum impact: font swap, color cleanup, hover/active states, layout fixes, component replacements, state designs, then typography polish.

## Core Rules

"Work with the existing tech stack. Do not migrate frameworks or styling libraries." Never break functionality, verify dependencies before importing libraries, and make focused, reviewable improvements rather than complete rewrites.
